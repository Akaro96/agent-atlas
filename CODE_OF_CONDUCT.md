# Code of Conduct

## Our Standards

Examples of behavior that contribute to a positive environment:

- being respectful and constructive
- assuming good intent while still being clear
- giving actionable technical feedback
- keeping discussions focused on improving the project

Examples of unacceptable behavior:

- harassment or personal attacks
- doxxing, intimidation, or threats
- deliberately hostile or degrading language
- publishing private information without permission

## Enforcement

Project maintainers may remove, edit, or reject comments, commits, code, wiki edits, issues, and other contributions that violate this Code of Conduct.

## Reporting

Use the security or support contacts defined in [SECURITY.md](./SECURITY.md) when appropriate, or open a maintainer contact issue if the matter is not security-related.

# Migration Guide

## Who This Is For

Use this guide if you are coming from:

- a giant public harness
- a personal dotfiles-style agent setup
- a single-agent setup you want to grow into a dual-agent workflow

## Migration Paths

### From a giant harness repo

Keep:

- the small number of commands or skills you actually use weekly
- any specialist prompts that genuinely improved outcomes

Do not port blindly:

- every command
- every agent
- every hook
- every “cool” workflow layer

Move in this order:

1. adopt the workspace structure
2. adopt the project-local instruction pattern
3. adopt the durable knowledge layer
4. only then re-add the few high-value extras

### From a dotfiles dump

Move:

- your reusable guidance into templates
- your personal defaults into global agent docs
- your durable notes into the vault

Do not keep:

- private history
- auth files
- machine-specific state

### From one agent only

Start with the runtime you already use.

Then add the second one only when:

- you know why you need it
- you have a real review or workflow reason

## Recommended First 30 Minutes

1. install the kit
2. read `START HERE.md`
3. create one real project
4. run the workspace doctor
5. create one vault draft note
6. compile the vault once

That is enough to understand the entire model.

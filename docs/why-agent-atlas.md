# Why Agent Atlas

## Short Answer

Because most people do not need the largest harness.
They need the clearest one that still delivers every day.

## The Case For This Project

### 1. Bigger is not always better

Many public harness repos win attention through:

- huge command surfaces
- giant skill counts
- complex framework shells

That is impressive, but it can also become maintenance debt.

Agent Atlas is intentionally narrower:

- fewer moving parts
- clearer install story
- stronger Windows fit
- real validation

### 2. The map is the product

This project treats these as first-class:

- the workspace root
- the project root
- the path-local override
- the durable knowledge layer

That means the system stays understandable even after months of use.

### 3. It supports mixed-tool reality

A lot of people do not live in one agent ecosystem forever.

This kit is stronger than many competitors if you want:

- Codex
- Claude Code
- optional Obsidian

without turning your setup into a giant compatibility experiment.

### 4. It values proof over vibes

This repo includes:

- a validator
- smoke tests
- a release bundle flow
- portable helper tools

That makes it easier to trust and easier to maintain.

## When This Is The Wrong Choice

Pick a bigger framework if you explicitly want:

- the broadest workflow operating system
- maximum command surface
- dozens or hundreds of built-in specialists
- more ceremony and more automation layers

## When This Is The Right Choice

Pick Agent Atlas if you want:

- a serious daily-driver harness
- cleaner boundaries
- less maintenance weight
- clearer adoption and migration
- a stronger Windows story

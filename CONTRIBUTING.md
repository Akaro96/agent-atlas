# Contributing

## Principles

- Keep the kit modular.
- Keep the root small.
- Prefer examples and validators over prose-heavy promises.
- Do not add private or host-specific state.

## Before Opening a PR

1. Run `pwsh -File .\scripts\Validate-AgentWorkspaceKit.ps1`
2. Check for accidental personal paths or usernames.
3. Keep changes scoped to one concern when possible.

## Good Contributions

- better portable templates
- clearer Windows guidance
- stronger validation
- better sample project docs
- optional modules that do not bloat the default install

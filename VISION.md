# Vision

## Public Name

**Agent Atlas**

Repository slug:

**agent-atlas**

## Why This Name

The core idea of the project is not “more magic”.
It is that agent behavior should have a clear **atlas**:

- mapped to the nearest project
- mapped through clear instruction layers
- mapped to durable knowledge when needed
- mapped through verification instead of improvisation

The public name is memorable.
The slug matches it directly.

## Positioning

Agent Atlas is for people who want:

- a serious daily-driver workspace harness
- both Codex and Claude Code support
- a separate knowledge layer instead of one giant instruction file
- stronger verification than most personal harness repos
- a clean Windows-first story

It is not trying to be the biggest “workflow OS” on GitHub.
It is trying to be the most credible lean harness in its class.
